Vocabulary for the representation of data on grants awarded by a municipality: Description back to ToC

The following figure shows the class diagram of the ontology, where it can be seen that the central element is the class [essubv:Convocatoria](http://vocab.ciudadesabiertas.es/def/sector-publico/subvencion#Subvencion), defined in this vocabulary:
The diagram goes around the main class [essubv:Convocatoria](#Convocatoria). This class is going to be related to [org:Organization](https://www.w3.org/ns/org#Organization), which is the class that will store both the beneficiaries of a grant, as well as the financing entities, areas or services that publish the call. Individuals are treated as organisations. The relationship between [essubv:Convocatoria](#Convocatoria) and [org:Organization](https://www.w3.org/ns/org#Organization) participant is made directly in the case of being [essubv:entidadFinanciadora](#entidadFinanciadora), [essubv:area](#area) or [essubv:servicio](#servicio) or through the class [essubv:Concesion](#Concesion) in the case of being a [essubv:beneficiario](#beneficiario). Each [essubv:Convocatoria](#Convocatoria) can be [essubv:gestionadoPor](#gestionadoPor) either by an [org:Organization](https://www.w3.org/ns/org#Organization) or by a [esadm:Distrito](http://vocab.linkeddata.es/datosabiertos/def/sector-publico/territorio#Distrito) [essubv:Convocatoria](#Convocatoria) is composed of the following attributes: - [dct:identifier](http://purl.org/dc/terms/identifier), text string that will store the call identifier. - [dct:title](http://purl.org/dc/terms/title), text string that will store the title of the call. - [essubv:objeto](#objeto), text string that will store the purpose of the call. - [essubv:nominativa](#nominativa), boolean that will indicate if the call is nominative or not. - [essubv:basesReguladoras](#basesReguladoras), URI that will store the link to the bases that will regulate the call. - [essubv:importePresupuestado](#importePresupuestado), float type that will store the value of the amount budgeted for the call. This amount will correspond to the total that is initially planned to be awarded to all beneficiaries. - [essubv:importeTotalConcedido](#importeTotalConcedido), float type that will store the value of the actual amount spent in the call among all the beneficiaries. This amount does not have to correspond to the one initially budgeted. - [essubv:fechaAcuerdo](#fechaAcuerdo), date type field that will store the date on which the call is agreed. - [espresup:anioFiscal](http://vocab.ciudadesabiertas.es/def/hacienda/presupuesto#anioFiscal), year type field that will store the fiscal year (period between January 1st and December 31th) which the call will be allocated to. - [essubv:programaAsText](#programaAsText), text string that will store the expense program classification. A call or a grant can have multiple classifications that will be stored as a single string, differentiated with a separator. This property will be deprecated when implementing classification as a SKOS. - [essubv:gastoAsText](#gastoAsText), text string that will store the economic classification of expense. A call or a grant can have multiple classifications that will be stored as a single string, differentiated with a separator. This property will be deprecated when implementing classification as a SKOS. Finally, the class [essubv:Convocatoria](#Convocatoria) can implement a [esconv:Convenio](#Convenio) or be implemented by it. In addition, several SKOS concept schemes have been used in this vocabulary to represent: - Types of instrument - Types of procedure - Spending program classification - Economic classification of expenditure - Grant topics In Figure 1, several relationships with the SKOS concept class can be observed, however, the specific values have not been specified as they are different depending on the taxonomy they represent. For example, the property [essubv:tipoProcedimiento](#tipoProcedimiento) represents whether the grant is of type "concesion-directa" or "concurrencia". The model also includes the class [essubv:Concesion](#Concesion) which will store all the data related to each grant award. This class is directly related to each [essubv:Convocatoria](#Convocatoria) with its beneficiary [org:Organization](https://www.w3.org/ns/org#Organization). All the information of the beneficiaries can be obtained through the property [essubv:beneficiario](#beneficiario), while all the information related to the call can be obtained through the property [essubv:convocatoria](#convocatoria). [essubv:Concesion](#Concesion) is composed of the following attributes: - [dct:identifier](http://purl.org/dc/terms/identifier), text string that will store the identifier of the grant. - [dct:title](http://purl.org/dc/terms/title), text string that will store the name of the grant project. - [essubv:importeSolicitado](#importeSolicitado), float type that will store the value of the amount initially requested by an [org: Organization](https://www.w3.org/ns/org#Organization). As indicated above, individuals are treated as organisations. - [essubv:importeConcedido](#importeConcedido), float type that will store the value of the amount finally granted to an [org: Organization](https://www.w3.org/ns/org#Organization). - [essubv:fechaSolicitud](#fechaSolicitud), date type field that will store the date the grant is requested. - [essubv:fechaConcesion](#fechaConcesion), date type field that will store the date the grant is awarded. - [essubv:programaAsText](#programaAsText), text string that will store the expense program classification. A call or a grant can have multiple classifications that will be stored as a single string, differentiated with a separator. This property will be deprecated when implementing classification as a SKOS. - [essubv:gastoAsText](#gastoAsText), text string that will store the economic classification of expense. A call or a grant can have multiple classifications that will be stored as a single string, differentiated with a separator. This property will be deprecated when implementing classification as a SKOS.
